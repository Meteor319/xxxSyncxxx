## 5. 策略引擎規格

### 執行模型
- 信號產生日 D → 當日收盤執行（可切換隔日開盤）
- 無未來資料洩漏：策略只能看到 ≤ D 的資料

### 交易費率
| 方向 | 費率 |
|------|------|
| 買入 | 手續費 × 0.6折 = 0.0855% |
| 賣出 | 手續費×0.6折 + 交易稅 0.3% = 0.3855% |

### 倉位管理
- 等權重分配（每個持倉佔 1/N 資金）
- 最大持倉數可設定（預設 10 檔）
- 交易單位：整張（1000股）

## 7. AI 分析模組

### 7.1 個股分析（`analyze.py` + `/stock` skill）

- 觸發：`python analyze.py {代號} [日期] --prompt-only` → 存 `analysis_data/{stock}_{date}.txt`
- Claude Code 讀取 txt，輸出完整分析報告並自動存檔至 `Report/{stock}_{date}_report.md`
- 報告章節：

```
## 📋 重點摘要（3~5條bullet：多空判斷/核心策略/關鍵籌碼/注意事項）
## 📝 操盤心法（先給結論）
## 📊 多空判斷（七段式評級 + 一句核心理由）
## 🔍 綜合分析
   技術面（均線/RSI/MACD）、量價關係、三大法人籌碼（表格）
   主力分點（近10日：淨買張數/均買賣均價/連續天數/集中度 + 今日進出解讀）
   月營收趨勢（近12個月 MoM/YoY/累計）
## 💡 操作策略（進場/停損/目標/報酬風險比）
   ### 隔日開盤應對（四種情境：跳空大漲/平開偏強/走弱翻黑/摜破大跌，各附具體價位）
## ⚠️ 主要風險（至少3點）
```

### 7.2 每日盤勢報告（`daily_report.py` + `/daily` skill）

- 觸發：`python daily_report.py [日期] --prompt-only` → 存 `analysis_data/daily_{date}.txt`

## 11. 獲利保護機制 (Profit Protection Mechanism)

### 11.1 需求背景

波段交易時，從進場到最高獲利點的過程中，往往伴隨價格回檔。需要一套自動化機制，在獲利開始回吐時發出減碼訊號，保護已實現的利潤。同時避免被主力洗盤觸發虛假訊號。

**案例**：3037 從進場點 880 上漲至 911（最高獲利 14 萬），隨後連跌至 821，回吐幅度 10%。需要在關鍵價位自動提醒減碼。

### 11.2 三層止損架構

```
進場點 ──→ 獲利設定 ──→ 第一層：浮動止損 ──→ 第二層：破位止損 ──→ 第三層：籌碼止損 ──→ 第四層：絕對止損
```

| 層級 | 觸發條件 | 動作 | 優先級 |
|-----|--------|------|--------|
| **第一層** | 獲利 < 最高獲利 × P1% | 減碼 R1% | 🟡 中 |
| **第二層** | 收盤 < MA20/MA5 | 減碼 R2% | 🟡 中 |
| **第三層** | 法人連 N 日賣超 + 量比 > Q | 減碼 R3% | 🟡 中 |
| **第四層** | 虧損 > L% | 全部清出 | 🔴 高 |

### 11.3 可調參數

#### 第一層：浮動獲利保護止損 

```yaml
tier1_floating_stop:
  profit_threshold: 0.70          # 保護門檻 % (50~90%)
  reduce_ratio: 0.50              # 減碼比例 (30~70%)
  split_reduce: false             # 是否分段減碼
  split_count: 2                  # 分段次數
```

**邏輯**：
- 追蹤最高獲利額度（在波段中動態更新）
- 當日獲利 < 最高獲利 × profit_threshold 時觸發
- 一次減碼 reduce_ratio 部位
- 若 split_reduce=true，分多次減碼，每次 reduce_ratio/split_count

#### 第二層：技術面破位止損

```yaml
tier2_technical_break:
  ma_types: [MA5, MA20]           # 監控的均線
  break_type: "close"             # close 或 intraday
  break_allowance: 0.0            # 容限百分比 (0~3%)
  reduce_ratio: 0.25              # 減碼比例
```

**邏輯**：
- MA5（短線敏感）或 MA20（中線穩健）破位時減碼
- break_type="close" 要求收盤破位（更穩定）
- 允許短暫跌破 allowance 百分比，防止洗盤假訊號

#### 第三層：籌碼面加速止損

```yaml
tier3_chipset_acceleration:
  targets: ["外資", "主力分點"]   # 監控對象
  consecutive_days: 3             # 連續賣超日數 (2~5)
  volume_ratio: 1.2               # 量比門檻 (0.8~1.5)
  watch_list: ["元大", "富邦"]   # 重點監控分點
  reduce_ratio: 0.25              # 減碼比例
```

**邏輯**：
- 外資連續賣超代表大資金認為需回調
- 加上量比確認，防止清淡盤虛假訊號
- 若監控對象是特定分點（如元大、富邦），當其連續減碼時加速訊號

#### 第四層：絕對止損

```yaml
tier4_absolute_stop:
  loss_limit: 0.10                # 虧損比例限制 (5~20%)
  trigger_on_loss: true           # 是否啟用虧損止損
```

**邏輯**：
- 最後防線，保護本金
- 虧損超過進價 × loss_limit 時全部清出

### 11.9 移動停利模組 — 均線追蹤停利 (MA Trailing Stop)

**設計理念**：隨股價上漲，均線本身自然向上爬升 → 停利線自動追蹤；跌破即停利出場。

#### 觸發邏輯

```
進場 → 獲利達 activation_profit% 啟動追蹤
     → 每日重算 MA{period} 作為停利線
     → 收盤跌破 MA × (1 - 容限%) 連續 confirm_days 天 → 觸發出場
```

#### 三種預設方案

| 方案 | 監控均線 | 容限 | 啟動門檻 | 確認天數 | 風格 |
|------|---------|------|---------|---------|------|
| **aggressive** | MA5 | 0.0% | +2% | 1日 | 短線敏感 |
| **balanced** | MA10 | 0.5% | +3% | 1日 | 中庸推薦 |
| **conservative** | MA20 | 1.5% | +5% | 2日 | 波段寬鬆 |

#### 3037 案例驗證（用 5/4 高點 911 模擬持倉）

| 預設 | 監控均線 | 當日停利線 | 觸發門檻 | 5/8 是否破位 |
|------|---------|----------|---------|------------|
| aggressive | MA5 | 852.20 | 852.20 | 收盤818<852 → 破位 ✅ |
| balanced | MA10 | 788.30 | 784.36 | 收盤818>784 → 安全 |
| conservative | MA20 | 775.09 | 763.46 | 未啟動（獲利3.5%<5%） |

**BALANCED 完整軌跡**：5/4 高點 911 持倉 → MA10 停利線從 788 隨股價上揚至 877（5/14）→ 5/15 收盤 821 < 門檻 873 → 觸發停利

#### 與其他層級的差異

| 層級 | 監控對象 | 觸發判準 | 動作 |
|------|---------|---------|------|
| 第一層浮動止損 | **獲利金額**回吐 | 獲利 < 最高獲利 × 70% | 減碼 50% |

### 12.3 五個核心指標（依優先順序）

#### 🥇 第一名：法人乖離訊號（最重要）

**核心概念**：股價方向 vs 外資方向的對應關係

| 股價 | 外資 | 訊號 | 解讀 |
|------|------|------|------|
| 大漲 | 大買 | 🟢 健康 | 法人共識買進 |
| 大漲 | 小買 | 🟡 警戒 | 法人不太認同 |
| **大漲** | **賣超** | 🔴 **拉高出貨** | 散戶推漲，法人賣 |
| 小跌 | 大買 | 🟢 撿便宜 | 法人逢低布局 |
| **小跌** | **大賣** | 🔴 **掩護出貨** | 小跌掩飾大賣 |
| 大跌 | 大買 | 🟡 試底 | 可能反彈訊號 |

**程式化定義**：
```python
def detect_institutional_divergence(price_chg_pct, foreign_net):
    """法人乖離訊號偵測"""
    if price_chg_pct > 3 and foreign_net < 0:
        return "RED_PUMP_AND_DUMP"  # 拉高出貨
    if -2 < price_chg_pct < 0 and foreign_net < -3000:
        return "RED_COVERED_SELLING"  # 掩護出貨
    if price_chg_pct > 1 and foreign_net > 1000:
        return "GREEN_HEALTHY"  # 健康多頭
    return "NEUTRAL"
```

### 13.9 歷史回測驗證計劃

**目標**：用 2024-2026 全市場資料驗證每種酒田型態的勝率與穩定性。

#### 驗證方法

```python
# 驗證流程偽碼
for stock in all_stocks:
    for date in 2024-01-01 to 2026-05-15:
        patterns = sakata.detect_all(stock, date, lookback=5)
        for pattern in patterns:
            # 記錄訊號日 + N 日後的價格變化
            for N in [1, 3, 5, 10, 20]:
                future_return = (price[date+N] - price[date]) / price[date]
                log_result(pattern, future_return, score)
```

#### 驗證指標

| 指標 | 目標 |
|------|------|
| **方向勝率** | 轉空型態 N 日後價格下跌的機率 ≥ 55% |
| **強度有效性** | 9-10 分訊號的勝率顯著 > 3-4 分訊號 |
| **籌碼加成驗證** | 含籌碼確認的勝率 > 無籌碼確認的勝率 ≥ 10% |
| **誤判率** | S 級訊號的反向風險 ≤ 20% |
