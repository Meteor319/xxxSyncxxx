這份規格說明書 (System Specification Document) 將完整記錄我們的共識，你可以直接將其存入 Gitea 作為專案的啟動文件。📄 AI 即時通報系統：資源調度特工 (Resource Agent) 規格說明書1. 專案概述 (Project Overview)本系統旨在解決 IC 設計產業中，Timecard (實際工時) 與 Workload (計畫工時) 數據破碎、難以即時對齊的痛點。透過 AI Agent Skills 模組化架構，系統將自動完成從 Excel 數據攝取、精密診斷到「人話版」調度建議的閉環流程。2. 系統架構 (System Architecture)採用 「確定性計算 (Python) + 語義推理 (LLM/Copilot)」 的雙引擎架構，確保數據 100% 正確，同時具備智慧化決策能力。數據中台： MariaDB (星狀模型設計)。邏輯定義： CLAUDE.md (作為 Agent 的運作規則憲法)。核心引擎： Python 3.8 (負責 ETL 與 Anomaly JSON 生成)。前端控制台： Streamlit (提供主管互動介面與 ECharts 視覺化)。通報渠道： Email, SharePoint List, Wiki。3. 數據架構設計 (Data Schema)3.1 維度表 (Dimension Tables)表名關鍵欄位說明dim_employeeemp_id, name, skillset, team基礎員工資料與專長領域。dim_projectproj_id, proj_name, priority專案基本資訊與優先等級。3.2 事實表 (Fact Tables)表名來源頻率說明fact_workloadExcel (Quarterly)每季原始計畫工時分配。fact_timecardExcel (Weekly)每周實際報工數據紀錄。4. Agent Skills 模組定義系統由四個核心技能組成，各司其職：Skill_Ingest (攝取技能)：監控 Gitea/資料夾，將 Excel 自動轉換為標準化 SQL 數據。具備名稱校對 (Mapping) 功能，解決 Excel 欄位不一的問題。Skill_Analyze (診斷技能)：依據 CLAUDE.md 定義的公式（如 $D_p, U_i$）進行精確運算。輸出： 結構化 Anomaly JSON (包含異常代碼與可用資源)。Skill_Reasoning (推理技能)：將 JSON 餵給 AI (Copilot)，生成具備商務邏輯的調度建議。Skill_Dispatch (派發技能)：自動更新 SharePoint List 做歷史存檔。更新 Wiki 周報並發送 Email 給相關主管。5. 異常診斷規則 (CLAUDE.md 節錄)警報閾值 (Thresholds)專案偏離度 ($D_p$)： 實際工時超過計畫 25% 標記為紅色警報。人員負載率 ($U_i$)： 單人周工時 > 48小時 (120%) 持續兩周標記為過載。調度邏輯優先搜尋：具備相同 skillset 且 utilization < 80% 之成員。6. 管理控制台 (Streamlit Dashboard)提供主管單一操作窗口，具備以下功能：Excel 上傳區： 支持拖拽式數據導入。AI 診斷區： 透過 ECharts 顯示各專案風險熱力圖。決策確認區： 預覽 AI 生成的建議，按一下「✅ 發布」後執行全渠道通報。權限控管： 區分「管理者（可寫入數據）」與「普通主管（唯讀報告）」。7. 維運與安全性 (Maintenance & Security)版本控制： 所有代碼與診斷邏輯存於 Gitea，實現規則可追溯。數據安全： 內網部署，數據不離開公司伺服器；使用 Service Account 存取 SharePoint。擴充性： 預留 API 介面，未來可接入 Gitea Commit 或 Jira 數據。
